# J-League Audience Prediction / Jリーグ観客動員数予測モデル

## 1. プロジェクト概要
過去のJリーグデータ（チーム、天候、スタジアム情報など）に基づき、**「J1リーグにおけるスタジアム観客動員数を予測する回帰モデル」**を構築しました。
ビジネスにおける需要予測やリソース最適化（警備・在庫管理など）への応用を想定したプロジェクトです。

### タスク詳細
* **目的**: 過去のデータから将来の試合の観客数を予測し、その精度を競う
* **学習データ期間**: 2006年 〜 2017年
* **予測対象期間**: 2018年 および 2019年

## 2. 使用技術・アルゴリズム
* **言語**: Python
* **モデル**: Gradient Boosting (勾配ブースティング)
* **主なライブラリ**: pandas, scikit-learn 等

## 3. ディレクトリ構成とファイル説明

本リポジトリのファイル構成は以下の通りです。

### ソースコード
| ファイル名 | 説明 |
| --- | --- |
| `Preprocessing.py` | **前処理用プログラム**<br>データの結合、欠損値処理、特徴量エンジニアリング（座標データや連勝数の計算など）を実行します。 |
| `GradientBoosting.py` | **学習・予測モデル**<br>前処理済みデータを用いて勾配ブースティングによる学習を行い、予測結果を出力します。 |

### データセット（Input）
コンペティション等で提供された元データです。

* `train.csv`: 学習用データ（2006-2017年の試合情報）
* `test.csv`: 評価用データ（2018-2019年の予測対象試合情報）
* `ex_venue_information.csv`: スタジアムに関する詳細データ
* `holidays_in_japan.csv`: 日本の祝日データ
* `match_reports.csv`: 試合結果データ（出場選手、得点経過など）
* `pref_lat_lon.csv`: 日本の県庁所在地の座標データ（距離計算に使用）
* `team_pref.csv`: 各チームのホームタウン（都道府県）データ

### 生成ファイル（Output / Intermediate）
プログラムによって生成されるファイル群です。

* `train_preprocessed.csv`: 前処理・特徴量付与後の学習用データ
* `test_preprocessed.csv`: 前処理・特徴量付与後の評価用データ
* `team_standings_min.csv`: チームの連勝数や順位情報を集計した特徴量データ
* `predict.csv`: **最終成果物**（予測結果）

## 4. 特徴量エンジニアリングの工夫
単なる試合情報だけでなく、外部データを用いて以下の特徴量を生成し、精度向上を図りました。
* **地理的要因**: 座標データ(`pref_lat_lon.csv`)を用い、アウェイチームの移動距離やアクセスの利便性を考慮。
* **チーム状況**: 過去の対戦成績や連勝数(`team_standings_min.csv`)を算出し、チームの勢いを数値化。
